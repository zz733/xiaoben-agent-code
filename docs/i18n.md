# Paseo 国际化 (i18n) 方案

## 架构概览

```
┌─────────────────────────────────┐
│  Settings → Language 选择器      │  ← 用户入口
└──────────┬──────────────────────┘
           │ setLocale("zh-CN")
           ▼
┌─────────────────────┐  ┌──────────────────────┐
│  Renderer (React)   │  │  Main Process (Electron)│
│  I18nProvider       │  │  onLocaleChanged()     │
│  t("key") 查表      │  │  重建菜单/对话框        │
│  → React re-render  │  │  → Menu.setApplicationMenu│
└─────────────────────┘  └──────────────────────┘
```

## 如何切换语言

1. **设置页面**：Settings → Language → 选择语言
2. **即时生效**：Renderer 侧 React context 更新，组件自动 re-render
3. **主进程同步**：通过 IPC `paseo:i18n:setLocale` 通知 Electron 主进程重建菜单和对话框
4. **自动检测**：首次启动根据系统语言自动选择（`app.getLocale()` / `navigator.language`）

## 如何增加新语种（3 步）

### Step 1：创建翻译文件

**Desktop 主进程**：

```bash
cp packages/desktop/src/i18n/locales/en.json \
   packages/desktop/src/i18n/locales/ja.json
```

**Renderer**：

```bash
cp packages/app/src/i18n/locales/en.json \
   packages/app/src/i18n/locales/ja.json
```

翻译 JSON 中的 value（key 不动）：

```json
{
  "menu.file": "ファイル",
  "menu.edit": "編集",
  "dialog.cancel": "キャンセル"
}
```

### Step 2：注册语种

**Desktop** — `packages/desktop/src/i18n/locale.ts`：

```typescript
import ja from "./locales/ja.json";

const LOCALES: Record<LocaleId, LocaleEntry> = {
  en: { label: "English", data: en },
  "zh-CN": { label: "简体中文", data: zhCN },
  ja: { label: "日本語", data: ja as Record<string, string> }, // ← 新增
};

// 更新 LocaleId 类型
export type LocaleId = "en" | "zh-CN" | "ja";

// detectLocale() 中添加映射
if (base === "ja") return "ja";
```

**Renderer** — `packages/app/src/i18n/index.tsx`：

```typescript
import ja from "./locales/ja.json";

const LOCALES: Record<LocaleId, LocaleEntry> = {
  en: { label: "English", data: en },
  "zh-CN": { label: "简体中文", data: zhCN },
  ja: { label: "日本語", data: ja as Record<string, string> }, // ← 新增
};
```

### Step 3：完成

重新构建即可，Settings → Language 会自动出现新选项。

## 上游源码改动详情

i18n 对上游源码的修改仅涉及 **3 个文件**，改动极小。以下是每一处修改的精确位置和内容，方便上游合并后手动恢复或使用自动脚本。

### 文件 1：`packages/desktop/src/main.ts`

#### 改动 A：添加 i18n import（第 52-53 行附近）

```diff
  import { setupApplicationMenu } from "./features/menu.js";
+ import { setupI18n, patchApplicationMenu, patchDialogHandlers } from "./i18n/index.js";
```

**原理**：`patchApplicationMenu` 替代 `setupApplicationMenu`，在构建菜单时注入翻译后的 label。`patchDialogHandlers` 替代 `registerDialogHandlers`，翻译对话框默认值。

#### 改动 B：替换菜单和对话框初始化（第 751-766 行附近）

```diff
   applyAppIcon();
-  setupApplicationMenu({
+  setupI18n();
+  patchApplicationMenu({
     onNewWindow: () => {
       void createWindow().catch((error) => {
         log.error("[window] failed to create window from menu", error);
       });
     },
   });
   ensureNotificationCenterRegistration();
   if (await runDesktopSmokeIfRequested()) {
     return;
   }
   registerDaemonManager();
   registerWindowManager();
-  registerDialogHandlers();
+  patchDialogHandlers();
   registerNotificationHandlers();
```

**原理**：

- `setupI18n()` — 初始化 i18n 系统，注册 IPC handlers，检测系统语言
- `patchApplicationMenu()` — 和 `setupApplicationMenu()` 功能相同，但 label 通过 `t()` 翻译
- `patchDialogHandlers()` — 和 `registerDialogHandlers()` 功能相同，但 "Confirm"/"Cancel"/"OK" 通过 `t()` 翻译

```diff
  function RootProviders({ children }: { children: ReactNode }) {
    return (
      <QueryProvider>
        <SafeAreaProvider>
          <KeyboardProvider>
+           <I18nProvider>
              <PortalProvider>
```

## 上游合并后的恢复流程

### 方式一：自动脚本（推荐）

```bash
# 1. 拉取上游代码
git fetch upstream main
git merge upstream/main

# 2. 自动重新应用 i18n 补丁
bash scripts/apply-i18n-patches.sh

# 3. 验证
bash scripts/apply-i18n-patches.sh --check

# 4. 构建检查
npm run typecheck
npm run lint
```

### 方式二：手动恢复

如果自动脚本因上游结构变化失效，按上面「上游源码改动详情」中的 diff 逐一手动恢复。

---

## 自动补丁脚本

`scripts/apply-i18n-patches.sh` — 幂等设计，可重复运行：

- **`--check` 模式**：仅检测补丁是否已应用，不修改文件
- **默认模式**：自动应用所有缺失的补丁
- 每项补丁独立检测，已存在的不会重复插入
